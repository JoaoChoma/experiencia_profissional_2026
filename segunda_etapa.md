# Segundo Bimestre - Segunda Etapa

O segundo bimestre está organizado em 8 semanas, iniciando no dia 27/04. A seguir, apresentamos o cronograma detalhado das atividades, seguindo uma lógica estruturada para o desenvolvimento do projeto.

## Semana 1: 27/04
- Aplicação das correções e melhorias identificadas durante a semana de apresentação dos resultados do primeiro bimestre.
- Elaboração dos diagramas de sequência: um diagrama para cada caso de uso identificado.

## Semana 2: 04/05
- Continuação da elaboração dos diagramas de sequência, garantindo que todos os casos de uso estejam cobertos.
- Elaboração do diagrama de classes de entidade, definindo as principais entidades do sistema.

## Semana 3: 11/05
- Continuação da elaboração do diagrama de classes de entidade, refinando os detalhes conforme necessário.
- Elaboração do diagrama de entidade e relacionamento (DER), modelando as relações entre as entidades no banco de dados.
- Organização dos projetos em três componentes principais: front-end, back-end e banco de dados.
- Escolha das tecnologias para implementação:
  - Para o front-end, utilizaremos React; especificamente, optaremos pelo Next.js para uma melhor experiência de desenvolvimento.
  - Para o back-end, utilizaremos Node.js; optaremos pelo Nest.js para uma estrutura robusta e escalável.
  - Para o banco de dados, escolheremos um modelo relacional devido à complexidade dos relacionamentos no sistema, garantindo melhor desempenho. Optaremos pelo PostgreSQL como tecnologia específica.
- Sugestão: Criar repositórios no GitHub para o **front-end web**, **front-end mobile** e **back-end**, facilitando a colaboração e versionamento do código.

## Semana 4: 18/05
- Seleção de um caso de uso para implementação completa de ponta a ponta.
- Sugestão: Escolher um caso de uso que possa servir como demonstração do produto, ideal para apresentações a potenciais investidores.
- Início do primeiro ciclo de implementação, focando na integração entre front-end, back-end e banco de dados.
- Realização das instalações necessárias em cada repositório, assegurando que todos os integrantes trabalhem com o mesmo ambiente e conteúdo na branch principal.

## Semana 5: 25/05
- Segundo ciclo de implementação: Continuação do desenvolvimento, refinando funcionalidades e corrigindo eventuais problemas identificados.

## Semana 6: 01/06
- Terceiro ciclo de implementação: Foco em testes iniciais e integração de componentes.

## Semana 7: 08/06
- Quarto ciclo de implementação: Finalização das funcionalidades principais e preparação para apresentação.

## Semana 8: 15/06
- Apresentação do projeto: Demonstração dos resultados alcançados até o momento.
- Exibição de todos os documentos e diagramas elaborados durante o bimestre.
- Contabilização da quantidade total de tarefas definidas e executadas pelo grupo.
- Contabilização das tarefas atribuídas e executadas por cada integrante do grupo.
- Apresentação do código do front-end com as conexões ao back-end.
- Apresentação do código do back-end com as conexões ao banco de dados.
- Demonstração do caso de uso funcionando de ponta a ponta, mostrando a integração completa do sistema.
